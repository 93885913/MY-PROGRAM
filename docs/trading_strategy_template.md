# A股交易系统模板 v2

## 1. 系统定位

- 系统名称：A股主线趋势跟随系统
- 交易风格：中短线趋势交易
- 适用周期：
  - 观察周期：20个交易日
  - 持仓周期：2到15个交易日
- 不适用：
  - 纯超短打板
  - 纯长期基本面持有

核心逻辑：
1. 先判断市场是否允许积极交易
2. 再筛选最强板块
3. 再筛选板块内最强个股
4. 最后等待买点确认后执行

默认数据采集与查看规则：
- 默认以 `东方财富网页逐页查看` 为主，不以单独接口数字抓取替代网页检查
- 优先打开并查看 `指数页 -> 板块页 -> 个股页`
- 报告中的图形判断、强弱判断、涨停/分时结构判断，以网页实际展示内容为准
- 若使用接口数据，只能作为网页信息的辅助校验，不能替代网页观察结论
- 默认保留 Playwright 浏览器检查痕迹或截图，方便后续复盘

默认发布规则：
- 默认使用 `scripts/publish_to_github.ps1` 发布分析产物到 GitHub
- Windows 下优先双击根目录的 `publish.bat` 执行发布
- 发布范围默认限定为 `docs/`、`reports/`、`artifacts/` 与发布脚本本身
- 不默认发布工作区里的临时测试文件、无关素材或历史实验文件
- 使用方式：`powershell -ExecutionPolicy Bypass -File .\\scripts\\publish_to_github.ps1`

## 2. 大盘过滤

固定观察指数：
- 上证指数
- 深证成指
- 沪深300

判断目标：
- 近20个交易日趋势方向
- 当日强弱和阶段强弱是否一致
- 成交活跃度变化
- 主力或北向资金方向
- 市场风格归类

市场环境定义：
- 强市场：
  - 深证成指或成长指数走强
  - 板块主线清晰
  - 强势股持续性较好
- 中性市场：
  - 指数分化
  - 板块轮动较快
  - 只适合做主线前排
- 弱市场：
  - 三大指数同步走弱
  - 强势股冲高回落明显
  - 优先轻仓或空仓

市场风格标签：
- 权重市
- 题材市
- 轮动市
- 退潮市

执行原则：
- 权重市：优先做大成交中军
- 题材市：优先做龙头和高辨识度个股
- 轮动市：缩短持仓周期，降低预期收益，减少开仓次数
- 退潮市：降低仓位，减少出手次数，宁可错过不做硬打

## 3. 板块筛选

目标：筛选2到3个最强板块

固定规则：
- 先看近20日涨幅
- 再看近3日是否仍在增强
- 再看板块资金流或主力净流入
- 再看上涨家数与下跌家数的板块广度
- 再看热点新闻、热词、资金关注是否持续出现

入选条件：
- 20日涨幅位居前列
- 当日仍然保持强势
- 板块内不是只有单一个股上涨
- 有清晰龙头和跟风梯队

剔除条件：
- 只有单日脉冲，没有持续性
- 板块内只有一只孤龙
- 明显高位退潮
- 成交活跃度不足

板块评分建议：
- 20日涨幅：30分
- 近3日强度：20分
- 资金流：20分
- 板块广度：15分
- 市场热度：15分

新增环境判定规则：
- 当板块评分低于`70分`时，默认该板块不具备主升浪配置价值，只能按震荡博弈看待
- 当板块内涨停股次日平均溢价为负时，系统自动把市场标签切换为`轮动/退潮市`
- 当连续2个交易日没有板块评分高于`70分`时，系统默认进入轮动市，禁止用主升浪思路重仓追涨

## 4. 个股筛选

每个板块固定选3只股票。

优先级：
1. 龙头股
2. 中军股
3. 高辨识度跟随强股

个股筛选标准：
- 近20日涨幅位于板块前列
- 近3日仍有强度
- 成交额和换手率处于活跃状态
- 主力净流入或资金承接明显
- 股价位于20日区间中上部
- 趋势结构没有明显破坏

新增基底形态评估：
- 优先选择上涨前已经横盘整理`2个月以上`、近期刚放量突破平台的个股
- 优先选择突破后第一次或第二次回踩确认的个股
- 回避已经连续上涨数月、远离20日线和60日线的纯高位博弈股
- 回避最近10个交易日加速过度、但没有新的平台整理结构的个股

基底形态加分建议：
- 2个月以上箱体或平台整理后突破：`+15分`
- 突破后缩量回踩不破平台：`+10分`
- 连续上涨超过2个月且无平台换手：`-15分`
- 最近5日明显加速、乖离过大：`-10分`

剔除条件：
- 一字板过多，不具备可交易性
- 流动性差
- 频繁冲高回落
- 明显掉队于板块
- 存在突发公告风险

个股评分建议：
- 20日趋势强度：25分
- 量价配合：20分
- 资金面：20分
- 板块地位：15分
- 可交易性：10分
- 基底形态：10分

## 5. 买点规则

只允许三类买点：

### 5.1 突破买点

- 平台整理后放量突破
- 板块同步走强
- 个股不是单纯高开拉升
- 必须优先是“有基底的突破”，不是高位情绪硬顶

### 5.2 回踩买点

- 强趋势股回踩5日线或10日线
- 缩量回踩，止跌后再转强
- 更适合趋势中军或龙头二次上车
- 若回踩位置同时靠近前平台上沿，优先级更高

### 5.3 分歧转一致买点

- 前一日分歧明显
- 次日板块继续强势
- 个股重新放量走强
- 只适用于主线前排，不适用于后排补涨

禁止买点：
- 连续大涨后盲目追高
- 板块转弱时逆势硬做
- 缩量尾盘拉升
- 跌破核心趋势后抄底
- 板块评分低于`70分`时还按突破主升浪模式追涨

## 6. 仓位管理

基础规则：
- 单只个股初始仓位：10%到20%
- 单一板块总仓位：不超过30%到40%
- 总仓根据市场环境调整

建议仓位：
- 强市场：60%到80%
- 中性市场：30%到50%
- 弱市场：0%到20%
- 轮动市：20%到35%
- 退潮市：0%到15%

加仓原则：
- 只有浮盈才允许加仓
- 加仓必须基于新买点
- 单股最多加仓2次

轮动市特别规则：
- 单笔交易目标从“博主升浪”下调为“赚震荡差价”
- 单日新开仓不超过`1到2笔`
- 不做后排追涨，不做尾盘冲动单
- 更重视隔日兑现和小波段滚动，而不是长持幻想

禁止事项：
- 亏损摊薄加仓
- 满仓押单一板块
- 情绪化重仓翻倍

## 7. 卖点规则

### 7.1 止损

- 跌破入场逻辑立即减仓或离场
- 趋势股优先看5日线和10日线
- 单笔风险固定控制在可承受范围内

### 7.2 止盈

旧规则保留为辅助参考：
- 冲高放巨量但无法继续走强，可先兑现部分仓位
- 到达前高或压力区，可先减仓锁定利润
- 连续加速后次日不能转强，优先兑现
- 板块转弱时，先卖跟风股

新增主规则：移动止盈优先于主观猜顶

规则A：均线跟踪止盈
- 对正在主升浪中的龙头或中军，不预设固定压力位
- 收盘价不破`5日线`，原则上不主动卖出
- 若波动放大，可把观察线放宽到`10日线`，但只能用于龙头和强中军

规则B：利润回撤止盈
- 当浮盈超过`10%`后，系统进入保护利润模式
- 允许最大利润回撤`30%`，超过即分批止盈或全部止盈
- 例如最高浮盈达到15%，若回撤到10.5%以下，则触发保护性卖出

规则C：分级执行
- 龙头股：优先用`5日线/10日线跟踪 + 利润回撤止盈`
- 中军股：优先用`5日线跟踪 + 分批止盈`
- 跟风股：不恋战，冲高滞涨或板块转弱就兑现

### 7.3 持股管理

- 龙头股允许更大的波动容忍，但必须尊重均线跟踪规则
- 跟风股更适合快进快出
- 中军股适合波段持有，但破位必须执行纪律
- 任何个股一旦板块评分跌破`70分`且自身掉队，优先降低仓位

## 8. 每日执行清单

### 开盘前

- 今天市场环境属于强、中、弱哪一类
- 今天最强方向是否延续昨天主线
- 今天计划关注哪2到3个板块
- 每个板块计划看哪3只股票
- 是否有板块评分高于`70分`
- 是否存在“涨停次日平均溢价为负”的退潮信号
- 是否已按 `东方财富网页逐页查看` 完成指数页、板块页、个股页检查

### 盘中

- 板块是否继续增强
- 龙头是否带动中军与跟风
- 是否出现符合系统定义的买点
- 是否出现止损或止盈信号
- 当前市场属于主升浪、轮动市还是退潮市
- 网页分时与盘口结构是否支持原先判断，若不支持则优先修正结论

### 收盘后

- 今天交易是否顺势
- 今天盈亏主要来自选股、买点、卖点还是仓位
- 板块主线是否切换
- 明天应该进攻、收缩还是等待
- 今天是否有个股满足“基底突破优先”的标准

## 10. 系统优缺点

优点：
- 顺势而为，容易抓主线
- 先板块后个股，容错率高于纯个股交易
- 适合网页公开数据分析
- 容易标准化和复盘
- 引入移动止盈后，更有机会拿住真正的主升浪龙头
- 以网页逐页查看为主，更容易识别图形、分时强弱和情绪加速段

缺点：
- 本质仍然是右侧交易，对启动初期判断要求较高
- 板块轮动很快时，回撤仍可能变大
- 对卖点纪律要求很高
- 如果忽略基底形态过滤，仍容易在高位末端追涨

## 11. 后续优化方向

- 增加涨停结构和炸板结构分析
- 增加板块次日溢价统计
- 增加公告和事件风险过滤
- 增加基底形态自动识别
- 增加交易日志统计胜率、盈亏比、最大回撤

## 12. 一句话总结

这套系统不是单纯的选股方法，而是一套完整流程：

大盘过滤 -> 板块筛选 -> 个股打分 -> 基底形态过滤 -> 买点确认 -> 仓位管理 -> 移动止盈/止损执行 -> 复盘修正
